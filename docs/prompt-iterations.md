# Prompt Iteration Record

## Overview

This document records major prompt directions and how they changed during the project.

## Iteration 1: Broad concept framing

**Goal:** Turn the assignment into a workable solo project.

**Prompt direction:** Ask AI to help convert the crowdfunding assignment into a repository-based solo submission plan.

**Problem:** The result was too broad and still mixed multiple delivery modes.

**Change made:** Narrow the project to `方案 C`, a static website, and a single product concept.

## Iteration 2: Product selection

**Goal:** Choose a concept that would be easy to explain, visualize, and support with pledge tiers.

**Prompt direction:** Explore product directions suitable for a student crowdfunding page.

**Problem:** Some options sounded interesting but would require hardware proof, manufacturing detail, or unrealistic claims.

**Change made:** Choose `ClarityPad`, an AI-assisted study planner with a believable scope and clear user value.

## Iteration 3: Landing page writing

**Goal:** Draft persuasive English copy for the crowdfunding page.

**Prompt direction:** Generate a page structure based on the required rhetorical moves.

**Problem:** Early draft language sounded too polished and startup-like, which risked feeling dishonest for a course prototype.

**Change made:** Reframe the page as a student-led concept prototype with transparent scope and practical delivery promises.

## Iteration 4: AI use documentation

**Goal:** Show AI collaboration honestly instead of pretending that every asset was fully produced.

**Prompt direction:** Ask AI to help organize process evidence, contribution notes, and reflection writing.

**Problem:** The assignment requires AI-generated visuals, but the current implementation round explicitly excluded image and video generation.

**Change made:** Keep the repository submission-ready, add clear placeholder asset notes, and mark the missing AI media as future replacement items instead of fabricating them.

## Iteration 5: Interaction design

**Goal:** Add lightweight interactivity without using a full framework.

**Prompt direction:** Implement a simple pledge tier selector and motion substitute inside a static page.

**Problem:** The page needed to feel more like a crowdfunding site than a plain document.

**Change made:** Add clickable pledge cards, storyboard frames, and visual section rhythm with CSS and small JavaScript.

## Final Prompting Principles

- Keep claims realistic for a student prototype.
- Prefer transparent scope over fake completeness.
- Use AI for structure, iteration, and speed, not for invented evidence.
- Leave obvious `TBD` markers where real share links or future media still need to be added.
