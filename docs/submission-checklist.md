# Submission Checklist

Use this checklist before emailing or printing the final assignment package.

## Repository

- Confirm the repository is public
- Push all local files to GitHub
- Make sure the README opens correctly in the browser
- Verify all links inside the repository still work

## Crowdfunding Page

- Open the published page and test it on desktop width
- Open the published page and test it on mobile width
- Check that all sections are visible and readable
- Check that the pledge tier interaction still works

## AI Evidence

- Replace every `TBD` AI link with a real shared link
- Add screenshots or exported records if needed by the teacher
- Confirm the AI collaboration statement matches the actual workflow

## Reflection and Role Notes

- Confirm the solo contribution statement is clear
- Confirm the reflection text matches the assignment expectations
- If required, export or print a separate document version for submission

## Submission Items

- Public GitHub repository URL
- Public deployed page URL
- AI conversation links
- Role breakdown and reflection document
- Printed page if a paper copy is required
